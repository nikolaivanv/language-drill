# Implementation Plan

## Task Overview

11 atomic tasks, bottom-up order: shared utilities → API layer → CDK wiring → Next.js UI. Tasks 1–2 have no dependencies and can run in parallel. Tasks 5–6 build the same file sequentially. All tasks end with `pnpm typecheck && pnpm test` from the repo root.

## Steering Document Compliance

- New files are co-located with existing peers: `lib/target-cell-size.ts` next to `lib/cell-key.ts`, `schemas/pool-status.ts` next to `schemas/progress.ts`, `middleware/admin.ts` next to `middleware/auth.ts`, `routes/admin.ts` next to `routes/progress.ts`.
- Test files sit next to their module (project convention — no orphan test files).
- Drizzle ORM for all DB access; Zod for all boundary validation.
- No new npm packages required (all dependencies already in the monorepo).

## Atomic Task Requirements
- **File scope**: each task touches 1–3 related files maximum
- **Time box**: 15–30 minutes
- **Single purpose**: one testable outcome per task
- **Specific files**: exact paths listed below
